error LNK2019解决方法

发表于 2016-07-22 | 分类于 填坑记录

使用VS开发的时候有时会遇到error LNK2005 和error LNK2019。

比如：

error LNK2019: unresolved external symbol imp_CrtDbgReportW referenced in function “public: double & __thiscall std::vector<double,class std

MSVCRT.lib(MSVCR80.dll) : error LNK2005: _strtol already defined in LIBCMTD.lib(strtol.obj)

MSVCRT.lib(MSVCR80.dll) : error LNK2005: _strtol already defined in LIBCMTD.lib(strtol.obj)

这类错误只需要将属性->C/C++->代码生成->运行库设置为多线程调试DLL(/MDd)就OK了。

Win10下提取chrome扩展程序

发表于 2016-07-21 | 分类于 填坑记录

目前网上提取chrome拓展程序的方式或多或少的过时了，今天想提取chrome中的一个扩展程序，用了很多方式都没找到，后来只能找cheome的数据，网上的路径一般都是win7的。

  1. 找路径
    在win10下的C:\Users\username\AppData\Local\Google\Chrome\User Data\Default\Extensions目录里面，有你安装过的所有扩展程序的文件夹，但是里面的文件夹名都是乱码，需要进行第二步。

  2. 找想要提取的程序的id
    以Google Keep为例，在chrome网上应用店找到其链接，
    https://chrome.google.com/webstore/detail/google-keep-chrome-extens/lpcaedmchfhocbbapmcbpinfpgnhiddi?utm_source=chrome-app-launcher-info-dialog
    则lpcaedmchfhocbbapmcbpinfpgnhiddi为其id，然后根据此id去找相应的文件夹。其实还有一个笨方法，在翻不了墙的情况下也能用，就是根据版本号去找，每个文件夹中会有该扩展程序的版本号，基本上通过这个和chrome的对应就可以找到了。

  3. 使用
    可以用作学习使用，如果要在别的浏览器中使用的话，需要在打包为crx后再拖到浏览器就好了。
